go 语言类型推断

2019/1/25

执行之前动态确保一个结构体拥有某个方法

类型断言之后,只能使用接口里定义的方法
比如 Runner 实际上有 SetId 和 GetId 两个方法
但是 动态约束了 demo.(interface{SetId(int64)}) 之后
就只能使用SetId 这个方法了

package main

import (
    "fmt"
)

type Runner struct {
    Num int64
}

func (self *Runner) SetId(num int64) {
    self.Num = num
    fmt.Println(self.Num)
}

func (self *Runner) GetId() int64 {
    return self.Num
}

func main() {
    var demo interface{} = new(Runner)
    
    demo.(interface {
        SetId(int64)
    }).SetId(134)
}

利用类型断言,确定变量的类型

可以根据不同的类型,进行不同的操作

package main

import (
    "fmt"
)

func checkType(in interface{}) {

    switch value := in.(type) {
    case int:
        fmt.Println("int", value)
    case string:
        fmt.Println("string", value)
    default:
        fmt.Println("unknow", value)
    }
}

func main() {
    ccc := 134
    checkType(ccc)
    ddd := "133"
    checkType(ddd)
    eee := []string{"d", "d", "e"}
    checkType(eee)
}

函数作为参数传递

函数作为参数传入 并且 被调函数, 参数类型为 interface{}时
该函数是无法执行的
将其进行类型转换,即可执行

package main

import "fmt"

type Handler func(int, string, int) // 定义一个函数类型

func demo(a int, b string, c int) {
    fmt.Println(a, b, c)
}

func check(in interface{}) {
    in.(Handler)(112, "ccc", 222)
}

func main() {
    check(Handler(demo))
}

类型断言

x := "hello world"
if v, ok := interface{}(x).(string); ok { // interface{}(x):把 x 的类型转换成 interface{}
    fmt.Println(v)
}
